# 00 - Change Log（本次修订说明）

> Project: Smart Salary Engine（SSE）  
> Version: V1.2  
> Date: 2026-07-06

## 本次修订目标

把原有提纲级文档升级为 MVP 可开发基线，重点补齐：

1. 数据模型；
2. Excel Sheet/列识别规则；
3. 员工数据池与来源追踪；
4. 数据合并冲突策略；
5. 检查引擎异常等级与规则；
6. 工资公式配置、计算顺序、精度和舍入；
7. API 请求/响应/错误码；
8. 前端页面流程与状态；
9. 存储、认证、日志、并发、测试策略。

## 关键设计变更

- 新增 Salary Run 作为一次工资核算任务的核心对象；
- 新增 SQLite 作为 MVP 状态存储，避免重启丢数据；
- 员工姓名作为 MVP 唯一匹配键，用户保证同一任务内无重名；
- 如果发现同名员工，系统生成 BLOCK 异常并要求检查 Excel，不做合并/拆分确认；
- 字段值统一保存来源文件、Sheet、行、列、导入批次；
- HR 人工补录值允许作为正式数据，但必须保存人工来源、处理人、时间和原因；
- 检查异常分为 BLOCK/WARN/INFO；
- BLOCK 异常未处理前禁止工资计算；
- 工资公式采用 YAML 配置，金额使用 Decimal；
- 每次计算生成 calculation version，不覆盖历史；
- API 从简单端点升级为前后端可协作的完整契约；
- 前端按任务步骤条组织流程；
- MVP 按本机单机使用设计，不做多用户协同和复杂权限；
- 导出必须以工资主表为模板生成新文件，尽量保留原表样式、列宽、表头和格式；
- MVP 不计算个税。
