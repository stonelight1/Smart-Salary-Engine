# 09 - Frontend（前端设计）

> Project: Smart Salary Engine（SSE）  
> Version: V1.2  
> Status: MVP 可开发基线

---

## 1. 前端目标

前端不是简单上传下载页面，而是 HR 的工资核算工作台。

核心目标：

1. 让 HR 清楚知道当前任务处于哪一步；
2. 对自动识别结果提供人工确认入口；
3. 对异常进行集中处理；
4. 工资计算前明确阻断原因；
5. 工资结果可预览、可解释、可导出。

## 3. 页面路由

```text
/login                          登录
/                               首页/任务列表
/runs/create                    创建工资任务
/runs/:runId                    任务概览
/runs/:runId/import             导入中心
/runs/:runId/mapping            识别确认
/runs/:runId/data-pool          员工数据池
/runs/:runId/check              数据检查
/runs/:runId/issues             异常中心
/runs/:runId/calculate          工资计算
/runs/:runId/explain            工资解释
/runs/:runId/export             导出中心
/config                         配置查看 MVP 只读
```

## 4. 全局布局

### 4.1 顶部区域

- 系统名称：Smart Salary Engine；
- 当前用户；
- 退出登录。

状态颜色：

| 状态 | 展示 |
|---|---|
| 已完成 | 绿色 |
| 当前步骤 | 蓝色 |

## 7. 识别确认页

### 7.1 Sheet 确认

展示：

| Sheet 名 | 系统识别类型 | 置信度 | 表头行 | 状态 | 操作 |
|---|---|---:|---:|---|---|
| 工资表 | 工资主表 | 0.93 | 2 | 已确认 | 修改 |
| Sheet2 | unknown | 0.42 | - | 待确认 | 指定类型 |

操作：

- 修改 Sheet 类型；
- 指定表头行；
- 忽略 Sheet；
- 保存确认。

### 7.2 列映射确认

展示：

| 原始列名 | 系统识别字段 | 置信度 | 是否必填 | 操作 |
|---|---|---:|---|---|
| 基本薪资 | 基本工资 | 0.95 | 是 | 修改 |
| 绩效 | 绩效奖金 | 0.82 | 否 | 确认 |

操作：

- 修改标准字段；
- 标记不导入；
- 保存为当前任务映射；
- 可选保存为全局别名。

```text
stores/
├── authStore.ts       # 登录用户、token
├── runStore.ts        # 当前 salary run
├── importStore.ts     # 导入批次、Sheet、列映射
```

## 15. 前端错误展示

| 错误码 | 展示方式 |
|---|---|
| UNAUTHORIZED | 跳转登录 |
| FILE_TYPE_NOT_SUPPORTED | 上传组件错误提示 |
| EXCEL_PARSE_FAILED | 弹窗展示原因 |
| BLOCK_ISSUE_EXISTS | 跳转异常中心 |
| RUN_BUSY | 顶部提示当前任务处理中 |
| INTERNAL_ERROR | 统一错误页或弹窗 |

---

## 16. 前端验收标准

1. HR 能按步骤完成完整流程；
2. 每一步都有明确下一步按钮；
3. BLOCK 异常在页面上醒目提示；
4. 计算按钮根据状态正确禁用；
5. 字段来源可以查看；
6. 工资解释能看懂公式和来源；
7. 导出文件可以下载；
8. 页面刷新后能根据后端状态恢复。
