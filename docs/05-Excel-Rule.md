# 05 - Excel Rule（Excel 识别与数据合并规则）

> Project: Smart Salary Engine（SSE）  
> Version: V1.2  
> Status: MVP 可开发基线

---

## 1. 设计目标

Excel Rule 负责把用户上传的“不稳定 Excel”转换为系统可理解的标准数据。

核心目标：

1. 自动识别 Sheet 类型；
2. 自动识别列名；
3. 将原始单元格转换为标准字段；
4. 按姓名合并员工数据；
5. 保留所有字段来源；
6. 遇到不确定情况交给人工确认。

---

## 2. 支持文件范围

| 项目 | MVP 规则 |
|---|---|
| 文件格式 | `.xlsx` |
| 单文件大小 | 默认不超过 50MB，可配置 |
| Sheet 数量 | 不限制，但超过 50 个给 WARN |
| 行数 | 1 万行以内优先保障 |
| 表头行 | 默认前 10 行内自动探测 |
| 合并单元格 | 读取时保留原值，复杂合并表头需要人工确认 |
| 密码保护文件 | MVP 不支持 |
| 公式单元格 | 读取 Excel 缓存值；若无缓存值则 WARN |

---

## 3. Sheet 类型

| Sheet 类型 | code | 说明 |
|---|---|---|
| 工资主表 | salary_main | 包含员工和主要工资字段 |
| 考勤表 | attendance | 出勤、缺勤、请假、迟到等 |
| 奖金表 | bonus | 绩效、提成、奖金 |
| 扣款表 | deduction | 罚款、其他扣款 |
| 社保公积金表 | social_security | 社保、公积金个人扣款 |
| 补贴表 | allowance | 餐补、交通补、通讯补等 |
| 其他补充表 | supplement | 无明确类型但可提取字段 |
| 未知 | unknown | 无法识别 |

## 4. Sheet 自动识别策略

Sheet 识别由三部分组成：

1. Sheet 名关键词匹配；
2. 表头字段命中率；
3. 数据特征识别。

### 4.1 评分模型

```text
sheet_score = name_score * 0.4 + header_score * 0.5 + data_score * 0.1
```

| 分数范围 | 处理 |
|---|---|
| >= 0.85 | 自动确认 |
| 0.60 - 0.85 | 标记需人工确认 |
| < 0.60 | 标记 unknown |

### 4.2 Sheet 名关键词示例

```yaml
sheet_rules:
  salary_main:
    keywords: ["工资", "薪资", "工资表", "薪资表", "应发", "实发"]
    required_fields: ["employee_name"]
  attendance:
    keywords: ["考勤", "出勤", "请假", "迟到", "缺勤"]
  bonus:
    keywords: ["奖金", "绩效", "提成", "奖励"]
  deduction:
    keywords: ["扣款", "罚款", "扣除"]
  social_security:
    keywords: ["社保", "公积金", "五险", "五险一金"]
```

## 5. 表头行识别

系统默认扫描前 10 行，寻找最可能的表头行。

### 5.1 表头行评分

```text
header_row_score = known_field_hit_count / non_empty_cell_count
```

加分项：

- 包含“姓名/员工姓名/人员姓名”；
- 包含多个金额字段；
- 非空单元格数量大于等于 2；
- 下一行开始出现数据。

减分项：

- 行内大部分是标题文本；
- 行内只有 1 个非空单元格；
- 包含“制表人/日期/单位”等说明性内容。

### 5.2 失败策略

| 场景 | 处理 |
|---|---|
| 找不到表头 | BLOCK，要求人工指定表头行 |
| 多行表头 | WARN，默认使用最后一行作为字段名，允许人工确认 |
| 表头重复 | BLOCK/WARN，按字段重要性决定 |

## 7. 列名识别策略

列名识别采用：

1. 完全匹配；
2. 别名匹配；
3. 规范化后匹配；
4. 模糊匹配。

### 7.1 列名规范化

规范化规则：

- 去除前后空格；
- 全角转半角；
- 去除换行符；
- 去除常见符号：`：`、`:`、`（元）`、`(元)`；
- 英文字母转小写。

示例：

| 原始列名 | 规范化 |
|---|---|
| ` 基本工资（元）` | `基本工资` |
| `员工\n姓名` | `员工姓名` |
| `实发:金额` | `实发金额` |

### 7.2 匹配结果

| 置信度 | 处理 |
|---|---|
| >= 0.90 | 自动映射 |
| 0.70 - 0.90 | 需人工确认 |
| < 0.70 | 不映射 |

## 8. 数据类型转换

| 标准类型 | 转换规则 | 失败处理 |
|---|---|---|
| string | 转字符串，去除前后空格 | 空值按字段规则处理 |
| money | 转 Decimal，保留 2 位以上中间精度 | BLOCK |
| number | 转 Decimal 或 int | BLOCK/WARN |
| percent | 支持 `10%`、`0.1` | BLOCK |
| date | 支持 Excel 日期、字符串日期 | WARN/BLOCK |
| enum | 根据枚举值映射 | WARN/BLOCK |
| boolean | 是/否、true/false、1/0 | WARN |

金额转换规则：

- 空字符串视为 null；
- `-`、`—` 默认视为 null；
- `1,234.56` 转为 `1234.56`；
- `￥1234.56` 转为 `1234.56`；
- 非法金额产生异常。

---

## 9. 姓名匹配规则

### 9.1 姓名清洗

- 去除前后空格；
- 替换全角空格；
- 去除不可见字符；
- 不自动删除中间字符；
- 不做拼音匹配。

### 9.2 匹配原则

MVP 使用姓名作为唯一匹配键。用户保证同一工资任务内没有重名员工。

| 场景 | 处理 |
|---|---|
| 姓名唯一 | 自动匹配 |
| 同一文件内重名 | 生成 BLOCK 异常 |
| 多文件同名 | 视为同一员工并合并 |
| 姓名为空 | BLOCK |
| 姓名疑似错别字 | 不自动匹配，WARN |

### 9.3 重名处理

系统展示：

- 姓名；
- 来源文件；
- Sheet；
- 行号；
- 可辅助字段：部门、工号、手机号后四位等，仅用于人工排查，不参与自动匹配。

MVP 处理规则：

1. 系统发现同一任务内姓名重复时生成 BLOCK 异常；
2. 系统不提供重名合并/拆分确认；
3. HR 检查来源 Excel，修正后重新上传；
4. 如果确认某行无效，可通过异常处理忽略该行。

非重名员工生成当前任务内唯一的 `employee_ref_id`。

## 10. 多次导入与冲突策略

### 10.1 数据合并规则

| 原值 | 新值 | 处理 |
|---|---|---|
| 空 | 有值 | 自动补齐 |
| 有值 | 空 | 保留原值 |
| 有值 | 相同 | 保留原值，追加来源 |
| 有值 | 不同 | 生成冲突异常 |
| 无字段 | 新字段 | 新增字段 |

### 10.2 冲突等级

| 字段类型 | 默认等级 |
|---|---|
| 核心工资字段 | BLOCK |
| 普通工资字段 | BLOCK |
| 部门、备注等辅助字段 | WARN |
| 非计算字段 | INFO/WARN |

### 10.3 冲突处理方式

- 使用原值；
- 使用新值；
- 手动录入，作为正式数据保存；
- 忽略该字段；
- 忽略该员工。

所有处理必须记录 audit log。

## 11. 来源追踪

每个字段值必须保存：

```json
{
  "file_id": "file_001",
  "file_name": "7月奖金表.xlsx",
  "sheet_name": "奖金",
  "row_index": 15,
  "column_index": 4,
  "column_name": "绩效奖金",
  "import_batch_id": "batch_002",
  "source_type": "EXCEL",
  "raw_value": "1200",
  "normalized_value": "1200.00"
}
```

人工补录值使用 `source_type = "MANUAL"`，必须保存处理人、处理时间和原因。

---

## 12. 识别失败处理

| 失败类型 | 等级 | 处理 |
|---|---|---|
| Sheet unknown | WARN/BLOCK | 非关键表 WARN，主表 BLOCK |
| 表头识别失败 | BLOCK | 人工指定 |
| 姓名列缺失 | BLOCK | 人工映射或重新上传 |
| 必填字段缺失 | BLOCK | 导入补充表或人工补录；人工补录可作为正式数据 |
| 列名低置信度 | WARN | 人工确认 |
| 类型转换失败 | BLOCK/WARN | 按字段重要性决定 |

---

## 13. 验收标准

1. 能识别常见工资主表 Sheet；
2. 能识别姓名、基本工资、奖金、扣款等标准字段；
3. 低置信度识别结果可人工确认；
4. 所有字段值有来源；
5. 多次导入补充表能自动补齐；
6. 不同来源同字段冲突能被发现；
7. 重名员工必须生成 BLOCK 异常；
8. 原始 Excel 文件不被修改。
